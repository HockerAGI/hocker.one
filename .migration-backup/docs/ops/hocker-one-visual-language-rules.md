# Hocker ONE Visual Language Rules

## Decision

Hocker ONE must not expose technical command names in the visual product experience.

## Visual names

- Production smoke test -> Pulso de Produccion
- Security hardening -> Blindaje del Sistema
- Tenant RLS -> Aislamiento de Portales
- Supabase guardrails -> Candado de Base de Datos
- Owner API gate -> Control de Acceso Maestro
- Full validation -> Validacion Total HOCKER
- Ops command center -> Centro de Operaciones NOVA

## Rule

Technical names may exist internally in scripts, logs and maintenance documents.

The user-facing Hocker ONE interface must use clean executive language.

## AGI supervision rule

AGIs may monitor, detect, diagnose and report automatically.

Critical actions require explicit approval: production database changes, payment flows, betting balances, secret rotation and migrations.
