# HOCKER AGI — Canon Real Consolidado de 16 AGIs

Versión: Sprint 12.6C.1B

Estado: documento canónico funcional previo a APIs Memory Mirror.

Base: Supabase `agis` + fuentes internas + historial operativo del proyecto + corrección explícita de REVIA.

## Principio central

El ecosistema HOCKER opera como una red IA↔IA ganar–ganar. Cada AGI aprende desde su dominio, comparte conocimiento útil, evita repetir errores y recibe actualizaciones filtradas por Syntia, validadas por NOVA y protegidas por Vertx/Jurix/Numia cuando aplique.

# Memory Mirror por AGI

Cada AGI debe recibir un feed especializado, no una memoria genérica.

Campos obligatorios para la siguiente migración:

```text
agi_id
source_type
source_name
source_url
source_platform
source_hash
semantic_hash

brand_id

valid_from
expires_at
retention_tier
times_seen
last_seen_at
prevents_error
error_pattern
requires_owner_approval
```

## Reglas anti-duplicado

Syntia debe bloquear o fusionar:

```text
1. misma fuente repetida
2. mismo aprendizaje con otra redacción
3. memoria vieja sustituyendo memoria nueva
4. error resuelto reapareciendo como novedad
```

## Retención

```text
Hot memory: memoria activa y consultada
Warm memory: resumen vigente
Cold archive: evidencia histórica fuera de consulta diaria
```

## Regla final

Ninguna AGI recibe todo. Cada AGI recibe exactamente lo que corresponde a su función, riesgo, cliente, plataforma y contexto.
